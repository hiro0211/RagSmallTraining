## OQ-001: Cross-Encoder のレイテンシ許容範囲 — 解決済み

- 背景: ローカル Cross-Encoder（`cross-encoder/ms-marco-MiniLM-L-6-v2`）はCPU環境では20件のReranking に数百ms〜数秒かかる可能性がある
- これが未確定だと困る理由: ユーザー体感に直結する。許容レイテンシを超える場合、Cohere API への切り替えや取得件数の調整が必要
- [Question] CPU環境でのReranking レイテンシが1〜3秒程度になる可能性があるが、許容できるか？
- [Answer] 取得件数を20件→10件に削減してレイテンシを短縮する方針で決定（DEC-002）
